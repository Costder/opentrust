# Release Documentation

OpenTrust uses a few different release documents so the README does not become a dumping ground.

- [Changelog](changelog.md) is chronological and implementation-focused.
- [Release notes](release-notes.md) are user-facing summaries of meaningful releases.
- [Upgrade notes](upgrade-notes.md) explain migration work for developers and operators.
- [Patch notes](patch-notes.md) capture small fixes, hotfixes, and operational patches that do not need a full release narrative.

Root-level [../../CHANGELOG.md](../../CHANGELOG.md) is kept as the canonical repository changelog entry point for GitHub and package tooling.
