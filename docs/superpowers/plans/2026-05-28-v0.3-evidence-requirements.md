# v0.3 — Evidence Requirements Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a structured `evidence_block` to the passport schema and enforce that `security_checked` passports include scanner output, reviewer identity, commit hash, dependency snapshot, and signed attestation.

**Architecture:** `PassportBase` already has an `evidence: dict | None` field that stores arbitrary JSON. This plan: (1) defines a `SecurityEvidenceBlock` JSON Schema `$defs` entry in `passport.schema.json`; (2) types the `evidence` field in Pydantic as `SecurityEvidenceBlock | None`; (3) enforces at API level that `trust_status: security_checked` has a complete evidence block; (4) extends CLI `_semantic_errors` with the same enforcement. Prereq: v0.2 plan complete (security.schema.json in bundle).

**Tech Stack:** Python, JSON Schema (Draft 2020-12), Pydantic v2, pytest

- [ ] **Step 4: Add enforcement to `api/src/routes/passports.py`**

Add this function alongside `_check_permission_scope`:

```python
def _check_evidence_block(payload: "PassportCreate") -> None:
    """Require a complete SecurityEvidenceBlock when trust_status is security_checked."""
    if payload.trust_status.value not in {"security_checked", "continuously_monitored"}:
        return
    if payload.evidence is None:
        raise HTTPException(
            status_code=422,
            detail=(
                "trust_status 'security_checked' requires a complete evidence block. "
                "Provide scanner_output, reviewer_identity, commit_hash, "
                "dependency_snapshot, and signed_attestation."
            ),
        )
    # Pydantic already validated the structure — just confirm all required fields are present
    # (they will be if the model parsed without error, but be explicit for the error message)
    evidence = payload.evidence
    missing = []
    if not evidence.scanner_output:
        missing.append("scanner_output")
    if not evidence.reviewer_identity:
        missing.append("reviewer_identity")
    if not evidence.commit_hash:
        missing.append("commit_hash")
    if not evidence.dependency_snapshot:
        missing.append("dependency_snapshot")
    if not evidence.signed_attestation:
        missing.append("signed_attestation")
    if missing:
        raise HTTPException(
            status_code=422,
            detail=f"Incomplete evidence block — missing: {', '.join(missing)}"
        )
```

Add `_check_evidence_block(payload)` call in both `create_tool` and `update_tool`, after `_check_permission_scope(payload)`:

    _EVIDENCE_REQUIRED_KEYS = {"scanner_output", "reviewer_identity", "commit_hash",
                                "dependency_snapshot", "signed_attestation"}
    if trust_status in {"security_checked", "continuously_monitored"}:
        evidence = data.get("evidence")
        if not evidence:
            errors.append(
                "$.evidence: trust_status 'security_checked' requires a complete evidence block "
                "with scanner_output, reviewer_identity, commit_hash, dependency_snapshot, "
                "and signed_attestation"
            )
        else:
            missing_keys = _EVIDENCE_REQUIRED_KEYS - set(evidence.keys())
            if missing_keys:
                errors.append(
                    f"$.evidence: incomplete evidence block — missing: {', '.join(sorted(missing_keys))}. "
                    "All five fields are required for security_checked."
                )
